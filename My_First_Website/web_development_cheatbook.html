HTML key concepts

  • HTML docs are all plain text.
  • Web browsers only do what you tell them to do.
  • Structure is added to elements with tags.
  • Tags must be nested properly.
  • Attributes can add functions or meaning to elements

Formatting

tag description
<h1 - <h6> Header text. I wonder why there is only 6 levels? *warning, not meant for resizing text
<p> Paragraph. Any old blob of text I guess.
<i> Italicized but not emphasized text.
<b> Bolded but not strongly emphasized text
<p> Paragraph
<em> Emphasized text. Basically italicized but user agents may treat it differently
<strong> Strongly emphasized text. Basically bolded but user agents may treat it differently
<br> Line break. Since white space in unmarked text gets ignored you need stuff like this.
<style> CSS style tag. Can be used to apply a style inline to affect only specific things.

Tables

tag description
<table> Base of a table.
<foot> Foot of a table. Child of table.
<head> Head of a table. Child of table.
<tr> Table row. Child of table.
<td> Table data. Each one is a cell on the row it belongs to. Child of tr.
<th> Table header. Just a header placed within a table.

Lists

tag description
<ul> Unordered list
<li> List item.
<ol> Ordered list.
<dl> Definition List.
<dt> Definition term. Child of dl and sibling of dd
<dd> Definition data. Child of dl and sibling of dt. Describes the dt it is paired with.

Sectioning

<html> Defines the root of an html document. kinda like main() in c++, though that can be confused with <main>
tag description
<head> Defines info about the document. This includes metadata and is also where you include your css and javascript files
<body> Defines the body of the page
<div> Generic division tag. Simply a grouping of elements. This can mean pretty much anything you want it to. Use id attributes and such to give them more meaning.
<article> Usually encloses the main body of text or content of a page. Up to you where an article starts and stops.
<section> Pretty much like a div. I've mostly seen it used to divide artcle content. Not very meaningful without an attribute describing it.
<aside> Side content like extra text blurbs or tables and such. This box is an aside!
<main> Main content. Whatever you want a user agent to see as the most important content on a page is this basically. Up to you what that is.
<header> Container for introductory content. Like the header of a word document basically. There are heads, headers and headings so don't get them confused
<footer> Extra content at the foot of a page. Usually copyright and contact info.

Linking

tag description
<meta> Metadata for the user agent.
<title> This is the little title on the browser tab.

Linking

<a> Anchor tag, in other words a Hyperlink tag. Not to be confused with <link> which is something else.
<img> Image link tag. Used kinda like a hyperlink but treated differently so it gets its own tag.
<link> Link to an external style sheet. Kinda like an anchor tag for css purposes

HTML vocabulary

HTML
HyperText Markup Language. I'm not going to forget that anytime soon, but there it is. If it can be read by a human without much coding knowledge, then its plain text. Markup refers to the tagging that HTML consists of basically.
CSS
Cascading Style Sheets. The language used to add flair and color to an HTML document. Essentially decides how things enclosed in tags are going to look.
Javascript
The language that makes magic happen. Things like animating the tabs on the quick reference box on this page are handled by Javascript. Some websites have Javascript do some a lot of the heavy lifting of the form and function of the site too. In general, you can expect most animated things on a website to be Javascript's responsibility, though it can do much more.
Plain Text
Text that is not computationally tagged, specially formatted, or written in code. HTML docs are apparently this, but it doen't look very plain to me. I'll ask someone later.
Elements
HTML elements consist of tags and the stuff within them. The tag itself defines what the element is and what it should do.
Attributes
Stuff added to tags to give them more meaning. Written as attribute="thing" with start tag.
Tags
<these> things. They basically do everything in html. They are basically markers that tell user agents and other programs what to do to the webpage.
